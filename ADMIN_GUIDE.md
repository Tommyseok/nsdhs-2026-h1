# 관리자 가이드 — 공지·일정·기도제목 등록

대시보드의 일정·공지·기도제목은 **GitHub 파일**에 저장되어 있습니다.
관리자가 직접 GitHub에서 파일을 수정하면 1~2분 후 모든 선생님 화면에 자동 반영됩니다.

## 📂 파일 3개

| 파일 | 용도 | 링크 |
|---|---|---|
| `data/schedule.json` | 일정 | [편집](https://github.com/Tommyseok/nsdhs-2026-h1/edit/main/publish/data/schedule.json) |
| `data/notices.json` | 공지 | [편집](https://github.com/Tommyseok/nsdhs-2026-h1/edit/main/publish/data/notices.json) |
| `data/prayers.json` | 다함께 기도제목 | [편집](https://github.com/Tommyseok/nsdhs-2026-h1/edit/main/publish/data/prayers.json) |

## ✏️ 수정 방법

1. 위 표에서 해당 파일의 **[편집]** 링크 클릭 (GitHub 로그인 필요)
2. 화면에서 파일 직접 편집
3. 페이지 맨 아래 **"Commit changes..."** 버튼 클릭
4. 그대로 **"Commit changes"** 한번 더 클릭
5. **1~2분 후** 모든 선생님 대시보드에 반영

## 📝 추가 예시

### 공지 추가 (notices.json)

`items` 배열의 **맨 위**에 새 객체 추가:

```json
{
  "items": [
    {
      "id": "N002",
      "date": "2026-06-08",
      "author": "교육국장",
      "title": "여름 수련회 신청 안내",
      "body": "안녕하세요 선생님.\n여름 수련회 신청서를 6/15까지 제출 부탁드립니다.",
      "active": true
    },
    {
      "id": "N001",
      "date": "2026-06-07",
      ...
    }
  ]
}
```

**핵심 규칙**:
- 콤마(`,`) 빠뜨리지 않기
- 따옴표(`"`)는 그대로 유지
- 줄바꿈은 `\n` 으로
- `id`는 N002, N003 ... 순서대로

### 일정 추가 (schedule.json)

```json
{"date": "2026-06-29", "title": "헌신예배", "note": "11시", "active": true}
```

### 다함께 기도제목 추가 (prayers.json)

```json
{
  "id": "P006",
  "date": "2026-06-08",
  "student": "학생이름",
  "cell": "셀 X",
  "teacher": "작성자",
  "text": "기도 내용...",
  "urgent": false,
  "active": true
}
```

`urgent: true` 면 대시보드에서 빨강 박스로 강조됩니다.

## 🗑 항목 삭제 / 숨김

**옵션 1**: 항목을 그대로 두고 `"active": false` 로 변경 → 화면에서 안 보임 (복구 가능)

**옵션 2**: JSON 배열에서 해당 객체 + 콤마 통째로 삭제 (영구 삭제)

## ❓ JSON 문법이 어렵다면

- 콤마 위치 헷갈리면 [JSONLint](https://jsonlint.com) 에 붙여넣어 검증
- 또는 저(관리자)에게 카톡으로 추가할 내용 보내면 제가 직접 처리

## 🔄 변경 후 확인

1. GitHub commit 후 **1~2분 대기**
2. dashboard 새로고침 → 반영 확인
3. 안 보이면: JSON 문법 오류 가능 → 위 JSONLint로 검증

# 📸 학생 사진 (사진 등록 페이지)

공지·일정과 달리 **사진은 GitHub commit이 필요 없습니다.** 선생님이 직접 올리면 끝.
사진 등록은 **전용 페이지(📸 사진 등록)** 에서만 합니다. (모든 페이지 상단 네비 바에 링크 있음)

## 올리는 법
1. 아무 페이지나 상단 네비 바에서 **📸 사진 등록** 클릭 (PIN + 본인 이름)
2. 본인 반 학생 카드가 뜸 (관리자는 전체 학생)
3. 학생 카드의 **카메라 버튼(📷)** 클릭 → 사진 파일 선택
4. 자동으로 정사각형으로 잘려 압축·업로드됨 → **즉시 저장**
5. 다른 선생님 화면에는 **1~2분 내** 반영

## 알아두기
- **권한**: 일반 선생님 = 본인 셀 학생, 관리자(국장단·목사·부장·총무) = 전체 학생.
- **사진 교체**: 같은 학생에 다시 올리면 덮어쓰기.
- **사진 없는 학생**: 성별에 맞는 기본 아바타(남=파랑, 여=분홍)가 자동 표시.
- **표시되는 곳**: 대시보드 내 반 · 출석 입력 · 전체 출석현황 · 학생상황+기도제목 · 학생 상세(셀편성). 이름 옆 동그란 사진을 **클릭하면 크게** 볼 수 있습니다.
- 공개용 편성표(학부모용)·관계도에는 사진이 표시되지 않습니다(개인정보 보호).
- 저장소: Supabase Storage (`student-photos` 버킷). 사진은 학생 케어 목적으로만 사용.

## 사진 삭제가 필요하면
화면엔 삭제 버튼이 없습니다(실수 방지). 삭제는 총무(석준원)에게 요청 → Supabase 대시보드에서 처리.

# 👑 관리자 페이지 (admin.html)

국장단·목사·총무(관리자 7명)만 접근. 대시보드 → **👑 관리자 페이지** 또는 `admin.html` 직접 접속 (PIN + 본인 선택, 관리자가 아니면 차단).

**한 화면에서 보이는 것**
1. **📊 접속 현황** — 선생님별 마지막 접속·접속 횟수, **아직 한 번도 안 들어온 선생님** 명단. (각 선생님이 페이지를 열면 자동 기록, 30분 단위)
2. **✅ 출석 입력 / 누락** — 주차를 골라 셀별로 *입력 완료 / 일부 / 미입력* 표시. **미입력 셀의 담임**이 바로 보여서 안내하기 좋습니다.
3. **📸 사진 등록 현황** — 전체 86명 중 등록 수·등록률, **아직 사진 없는 학생**을 셀별로.
4. **🙏 기도·📢 공지 현황** — 공유 기도제목/공지 개수 + GitHub 편집 바로가기.

**출석이 공유되는 원리**: 선생님이 `출석 입력`에서 저장하면 자동으로 중앙(Supabase)에 올라가고, 다른 선생님·관리자 화면에 함께 보입니다. (예전엔 각자 기기에만 저장됐는데 이제 공유됩니다)

> 데이터 초기화(학기 말 등)나 잘못된 기록 삭제는 총무에게 요청해 주세요.
